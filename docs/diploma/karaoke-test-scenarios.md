# Программа и методика испытаний подсистемы караоке

Документ предназначен для раздела «Тестирование» выпускной квалификационной работы по теме **«Приложение для визуализации музыки с анализом аудио»**.

## Цель испытаний

Проверить корректность работы подсистемы караоке-визуализации при автоматическом и ручном поиске синхронизированного текста песни, обработке неполных метаданных аудиофайлов и отображении текста во время воспроизведения.

## Тестовое окружение

- Desktop-приложение Loomi (Tauri / web-клиент).
- Активный интернет-доступ к каталогу LRCLIB (https://lrclib.net).
- Набор тестовых аудиофайлов с различным качеством метаданных и имён файлов.

| 14 | Сбой сети | Трек с корректными метаданными, отключённый интернет | Выполнить автоподбор или открыть альтернативы | Приложение сообщает о недоступности LRCLIB, не зависает и не ломает воспроизведение | |

## Критерии успешного прохождения

- Не менее **12 из 14** сценариев выполняются без ошибок.
- Критические сценарии **1, 4, 5, 8, 10** должны быть выполнены обязательно.
- При ошибках поиска пользователь получает понятное сообщение и альтернативный способ загрузки текста.

## Вывод для раздела «Результаты тестирования»

> Подсистема караоке-визуализации корректно обрабатывает аудиофайлы с полными, неполными и некорректными метаданными, поддерживает автоматический и ручной поиск синхронизированного текста, выбор альтернативных записей из каталога LRCLIB и отображение текста, синхронизированного с текущим временем воспроизведения. Результаты испытаний подтверждают работоспособность разработанной подсистемы и её пригодность для использования в составе desktop-приложения.
