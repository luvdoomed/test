# Облачная синхронизация Loomi (гибрид C)

## Модель

| Данные | Где хранится | Синхронизация |
|--------|--------------|---------------|
| MP3/FLAC | Только локально (AppData) | Нет |
| Метаданные библиотеки | Локально + PostgreSQL/SQLite на сервере | Да |
| Обложки | Локально + BLOB на сервере | Да |
| LRC | localStorage + сервер | Да |
| Настройки приложения | localStorage + сервер | Да |
| Пресеты визуализаторов | localStorage + сервер | Да |
| Пользовательские визуализаторы (.tsx) | AppData + сервер | Да |
| Аудио в облаке (опционально) | Сервер, лимит 500 МБ | API + кнопки в библиотеке |

## Компоненты

- **Клиент:** `src/services/cloudApi.ts`, `cloudSync.ts`, `store/authStore.ts`
- **Сервер:** `server/` — Express, SQLite, JWT

## Деплой API

См. [docs/deploy/vps.md](../deploy/vps.md) — VPS, nginx, HTTPS, `VITE_API_URL`.

## Запуск для разработки

```bash
# терминал 1
npm run server:dev

# терминал 2 — скопировать .env.example в .env
npm run tauri dev
```

## Сценарий «другое устройство»

1. Пользователь входит в аккаунт → `pullCloudSnapshot`.
2. В библиотеке появляются треки из облака (метаданные, обложки, LRC).
3. Воспроизведение возможно после повторной загрузки того же файла на диск (или кнопки «скачать из облака», если аудио было прикреплено).

## Клиентские сценарии (v2)

- **Привязка:** drag-and-drop MP3 с тем же именем/размером → запись из облака получает локальный файл.
- **Прикрепить к облаку:** иконка ↑ на треке с локальным файлом (лимит 500 МБ).
- **Скачать из облака:** иконка ↓ на треке «нужен локальный файл», если аудио есть в облаке.
- **Индикатор:** точка на иконке профиля (зелёная / жёлтая при sync / красная при ошибке).
